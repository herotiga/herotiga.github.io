React开发笔记

陆续记录React开发中需要注意的点。

函数组件和类组件

区别：函数组件开销小，类组件开销大（因为需要创建实例），对于有状态的组件可以采用函数组件+hooks的方式开发，

结论：只使用函数组件+hooks的方式，尽量不用类组件。

状态管理

state

state 用于保存渲染中的数据，类似一个快照，set这个state的时候会触发页面的重新渲染。

一个state变量的值在获取UI快照时就被确定了，如果要修改state则需要传入函数。

immer

使用 immer 代替state。

状态提升

如果我们需要在多个组件之间共享状态，可以通过将这个状态提升到父组件中，再将状态通过props传递给多个子组件。

事件

父组件通过props将事件处理函数传递给子组件处理

可以直接将事件处理函数卸载组件中，也可以将事件处理函数作为 props 传递给子组件，在子组件中处理。

事件传播

如果父元素和子元素绑定了同一个事件，触发子元素的事件后会一层一层传播到外层元素并触发事件。

阻止事件冒泡

方法：e.stopPropagation()

阻止事件默认行为

方法：e.preventDefault()

捕获子元素事件

可以在父元素上绑定 onClickCapture 事件来监听子元素的事件。
